## HAICD 的灵魂与核心（2026-03-26 深化）

### 核心定义

```
HAICD 不是"状态选择器"，而是"状态流的测量与制约系统"

智能体输出 = f(状态流，模型约束，资源约束)
```

### 关键原则

| 原则 | 说明 |
|------|------|
| **权重动态可调** | 意义评估权重随角色/场景动态调整，不预设固定值 |
| **状态流跟踪** | 安全/意义/演化/迭代均基于状态演化轨迹测量 |
| **三重制约** | 状态（初始/即时）+ 模型框架 + 资源（可及/可证/稳定） |
| **理论融合** | 理论 01（人）与理论 02（智能体）紧密关联，不可割裂 |
| **状态透明** | 用户不需知道当前状态（避免干扰），内置状态，刻意查询时可输出 |
| **反馈机制** | 短期=用户持续使用意愿；长期=文档 03 效果评价模型 |

### 状态流概念

```
状态流 = 任务进程中状态的时序演化轨迹

示例：
问题发现 → 文献调研 → 假设形成 → 实验设计 → 数据收集 → 数据分析 → 论文撰写

每阶段适配不同状态，形成完整状态流
```
